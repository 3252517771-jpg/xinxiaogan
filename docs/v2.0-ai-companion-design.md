## 一、版本定位

首版是"数据记录工具"，v2.0 升级为**有感知、会提醒、能建议**的智能健康陪伴系统。

核心能力三角，零新增路由、零新增表，全部通过**扩展现有 API 返回体 + 扩展已有表字段**实现：

```
           🛎️ 推送通知 (Server酱)
         /                      \
   用户数据 ————→  AI 能力三角
  （继续手动录入）  ├─ 🤖 ML 风险预测（scikit-learn 本地推理）
                   ├─ 🔍 规则引擎行为识别（if-else 阈值）
                   └─ 💬 DeepSeek API 个性化建议（Privacy Guard 脱敏）
```

## 三、模块详细设计

### 3.1 🤖 ML 风险预测

#### 3.1.1 选型

| 项目 | 选择 |
|------|------|
| 框架 | scikit-learn |
| 模型类型 | RandomForestClassifier（默认参数，不调优） |
| 导出格式 | `joblib` |
| 推理方式 | FastAPI 内 `joblib.load()` 直接推理，应用启动时加载一次 |
| 模型文件位置 | `backend/ml_models/risk_classifier.joblib` |
| 训练环境 | Conda `D:\conda-envs\claude-code` |

#### 3.1.2 数据集与训练

- 使用真实公开数据集训练（CDC NHANES 2017-2018 官方公开数据）
- 特征映射：systolic_bp, diastolic_bp, heart_rate, blood_glucose, waist_cm, cholesterol → 训练输入
- 目标：三分类（low / medium / high risk）

训练脚本放 `backend/ml_models/train.py`，生产不需要运行。

#### 3.1.3 推理接口（在现有 `POST /api/health/risk` 中实现）

```python
# backend/app/services/risk_predictor.py

import joblib
import os
from pathlib import Path

_model = None
_scaler = None

def load_models():
    global _model, _scaler
    model_path = Path(__file__).parent.parent.parent / "ml_models"
    _model = joblib.load(str(model_path / "risk_classifier.joblib"))
    _scaler = joblib.load(str(model_path / "risk_scaler.joblib"))

def predict_risk(features: dict) -> dict:
    """输入体征特征，返回风险等级"""
    if _model is None:
        load_models()
    # 特征顺序与训练时一致
    input_array = [[
        features["systolic_bp"],
        features["diastolic_bp"],
        features["heart_rate"],
        features["blood_glucose"],
        features["waist_cm"],
        features["cholesterol"],
    ]]
    scaled = _scaler.transform(input_array)
    pred = _model.predict(scaled)[0]        # 0=low, 1=medium, 2=high
    proba = _model.predict_proba(scaled)[0]  # 概率分布
    level_map = {0: "low", 1: "medium", 2: "high"}
    return {
        "risk_level": level_map[pred],
        "risk_probability": round(float(proba[pred]), 3),
        "risk_alert": pred >= 2,
    }
```

> 模型在 FastAPI 启动时由 `@app.on_event("startup")` 调用 `load_models()` 加载一次。

### 3.2 🔍 规则引擎行为识别

#### 3.2.1 行为模式定义（覆盖 5 个维度）

| ID | 模式 | 检测规则 | 维度 | 文案 |
|----|------|---------|------|------|
| late_night | 连续熬夜 | 连续 3 天入睡 > 01:00 | sleep | "近期熬夜较多，注意调整作息哦 🌙" |
| short_sleep | 睡眠不足 | 连续 3 天时长 < 6h | sleep | "睡眠时间偏短，争取早点休息 💤" |
| meal_skip | 三餐不规律 | 近 7 天缺餐 ≥ 4 次 | diet | "三餐记录不完整，记得按时吃饭 🍚" |
| sedentary | 久坐模式 | 近 7 天运动 < 3 次 | exercise | "近期运动偏少，出门走走活动一下 🚶" |
| high_stress | 高压模式 | 压力自评连续≥ 7 分 | stress | "最近压力较大，记得适当放松 🧘" |
| abnormal_sign | 体征异常 | 任意体征超出医学参考范围 | risk | "某项指标偏高，建议持续关注 👀" |

#### 3.2.2 实现

```python
# backend/app/services/behavior.py

BEHAVIOR_RULES = [
    {"id": "late_night", "label": "近期熬夜较多，注意调整作息哦 🌙", "dimension": "sleep",
     "check": lambda records: sum(1 for r in records if r["sleep_time"] > "01:00") >= 3},
    {"id": "short_sleep", "label": "睡眠时间偏短，争取早点休息 💤", "dimension": "sleep",
     "check": lambda records: ... },  # 时长 < 6h
    # ... 其余规则
]

def detect_behaviors(dimension: str, records: list) -> list[str]:
    """返回命中的行为标签列表"""
    return [rule["label"] for rule in BEHAVIOR_RULES
            if rule["dimension"] == dimension and rule["check"](records)]
```

> 在 `POST /health/{dimension}` 的 service 层调用，结果拼入返回体。

#### 3.3.1 Privacy Guard 脱敏层

```python
# backend/app/services/ai_advice.py

def anonymize_for_llm(data: dict, dimension: str) -> dict:
    """
    脱敏规则：
    1. ❌ 丢弃所有用户身份字段（nickname, age, gender, height, weight）
    2. ✅ 数值等级化（连续熬夜3天 → "近期"；评分75 → "中等偏上"）
    3. ✅ 趋势抽象化（具体数字 → "稳定/波动/下降/上升"）
    4. ✅ 无水印/无日期/无历史全量序列
    """
    return {
        "dimension": dimension,
        "score_level": _levelize(data.get("score", 0)),       # 优秀/良好/一般/偏低
        "sleep_time_desc": _describe_time(data.get("sleep_time")),  # 仅作息维
        "wake_time_desc": _describe_time(data.get("wake_time")),
        "quality_desc": _quality_text(data.get("quality")),
        "trend_desc": _trend_text(data.get("recent_scores", [])),
        "behavior_tags": data.get("behavior_tags", []),
    }
```

#### 3.3.3 API 调用

```python
import openai
from app.config import settings

client = openai.OpenAI(
    api_key=settings.DEEPSEEK_API_KEY,
    base_url="https://api.deepseek.com",
)

async def get_ai_advice(anonymized: dict) -> str:
    """调用 DeepSeek API 获取个性化建议"""
    try:
        response = client.chat.completions.create(
            model="deepseek-chat",
            messages=[
                {"role": "system", "content": SYSTEM_PROMPT},
                {"role": "user", "content": USER_PROMPT_TEMPLATE.format(**anonymized)},
            ],
            max_tokens=200,
            temperature=0.7,
        )
        return response.choices[0].message.content
    except Exception:
        # API 不可用时的 fallback
        return None
```

#### 3.3.4 安全与隐私

- DeepSeek API Key 通过 `.env` 注入，**不写入代码**
- 送入 LLM 的数据**仅包含脱敏摘要**，不含 username / age / height / weight / gender
- API 调用失败 → fallback 到规则模板建议，**用户无感知**
- `enable_ai_advice` 关闭 → 直接走规则模板，不调用 DeepSeek API
- 首次启用时弹隐私说明（Frontend Toast 一次）
