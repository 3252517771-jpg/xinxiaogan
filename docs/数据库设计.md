## 一、数据库选型

| 环境 | 数据库 | 说明 |
|------|--------|------|
| 开发 | SQLite | 零配置，直接文件数据库 |
| 生产 | **PostgreSQL 15+** | ECS部署，成熟稳定 |
| 缓存 | Redis（后评估） | 会话缓存/评分缓存 |

选型理由：
- PostgreSQL 与 Python FastAPI + SQLAlchemy 生态完美匹配
- JSONB 字段支持方便后期扩展（如 LayoutConfig 的样式参数）
- 全文检索支持未来历史记录搜索
- SQLite 开发期与 PostgreSQL 生产期通过 SQLAlchemy 抽象层无缝切换

## 三、各对象业务场景

### 3.1 User（用户账户）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 注册 | INSERT | username, password_hash | 用户提交注册表单 |
| 登录 | SELECT + JWT验证 | username, password_hash | 用户提交登录表单 |
| 切换管理员 | UPDATE | is_admin | 管理员密码验证通过 |
| 删除账户 | DELETE（级联） | — | 用户注销账户 |

### 3.2 UserProfile（个人资料）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 首次登录填充 | INSERT + 自动创建 | 所有字段设默认值 | 用户注册成功 |
| 编辑资料 | UPDATE | nickname, age, gender, height, weight | 用户在个人主页编辑 |
| 读取资料 | SELECT | 全部 | 登录后加载个人主页 |
| 影响评分 | 读取源数据 | age, gender, height, weight | DailyScore 计算时读取 |

### 3.3 SleepRecord（作息记录）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 录入睡眠数据 | INSERT | sleep_time, wake_time, sleep_quality, interruption_count, score | 用户点击[记录今日] |
| 同一天重复录入 | UPDATE（upsert） | 全部记录字段 | 用户当天再次提交 |
| 查看当日记录 | SELECT by user_id + date | 全部 | 进入作息页 |
| 历史记录查询 | SELECT by user_id + date_range/dimension | 全部 | 个人主页历史筛选 |
| 计算当日评分 | 内部计算 | → 更新 DailyScore.sleep_score | 提交后自动触发 |

**评分规则**：
- 入睡时间：23:00前+10分，23:00-24:00 +5分，24:00后 -5分
- 睡眠时长：7-9h +40分，6-7h +20分，<6h -10分
- 睡眠质量：⭐×20分（5⭐=100，1⭐=20）
- 中断次数：0次+20分，1次+15分，2次+5分，≥3次0分
- 得分 = 四维加权平均（v1简化版本）

### 3.4 DietRecord（饮食记录）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 录入饮食 | INSERT | meal_type, food_desc, calories, recorded_at | 用户点击[记录今日] |
| 同一天多餐次 | INSERT（允许多条） | 同一date不同meal_type | 用户记录多餐 |
| 查看当日饮食 | SELECT by user_id + date | 全部 | 进入饮食页 |

**评分规则**：
- 三餐完整度 +40分（早中晚均记录满分）
- 热量合理度（1600-2200 kcal）+40分
- 食物多样性（通过food_desc关键词简单判断）+20分

### 3.5 ExerciseRecord（运动记录）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 录入运动 | INSERT | exercise_type, duration_min, intensity, steps, heart_rate, score | 用户点击[记录今日] |
| 同一天多次运动 | INSERT（允许多条） | 同一date多次 | 用户多次运动 |
| 查询今日汇总 | SELECT SUM | duration_min, steps | 进入运动页 |

**评分规则**：
- WHO推荐运动量（每周150min中强度）：日达标30min +40分
- 强度权重：低1x / 中1.5x / 高2x
- 步数：>8000 +30分，>5000 +15分，<5000 0分
- 心率正常范围 +30分

### 3.6 StressRecord（压力/心理记录）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 压力自评 | INSERT | stress_level, anxiety_level, emotion_tag, recorded_at | 用户点击[记录今日] |
| 同一天重复 | UPDATE（upsert） | 全部 | 用户当天再次评价 |

**评分规则**（反向指标，越低越好）：
- 压力自评(1-10)：1-3 +40分，4-6 +20分，7-10 0分
- 焦虑自评(1-10)：1-3 +30分，4-6 +15分，7-10 0分
- 情绪标签正向（开心/平静/放松）+30分，中性0分，负向（焦虑/烦躁/低落/紧张）-10分

### 3.7 RiskRecord（体征/风险记录）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 体征录入 | INSERT | systolic_bp, diastolic_bp, heart_rate, blood_glucose, waist_cm, cholesterol, recorded_at | 用户点击[记录今日] |
| 同一天重复 | UPDATE（upsert） | 全部 | 用户当天再次测量 |

**评分规则**（按医学参考范围）：
- 收缩压：90-120 +15分，120-140 +10分，>140 0分
- 舒张压：60-80 +15分，80-90 +10分，>90 0分
- 静息心率：60-100 +15分，其他0分
- 血糖：3.9-6.1 +15分，其他0分
- 腰围：男<90/女<85 +20分，超过0分
- 胆固醇：<5.2 +20分，>5.2 0分

### 3.8 DailyScore（每日综合评分）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 提交任意维度数据后 | UPSERT | 对应维度score更新 + overall_score重算 | 任何维度数据变更后 |
| 进入首页 | SELECT by user_id + date | 全部 | 用户进入首页 |
| 查看趋势图 | SELECT by user_id + date_range（近7天） | overall_score + 各维度score | 首页/详情页加载 |
| 每日凌晨自动创建 | INSERT（默认值） | 所有score=null，overall_score=null | cron任务（post-MVP） |

**评分逻辑**：
```
overall_score = AVG(sleep_score, diet_score, exercise_score, stress_score, risk_score)
其中 stress_score 需翻转（压力越低分越高）
注：只有当天有数据的维度参与计算，无数据维度不纳入
```

### 3.9 LayoutConfig（布局配置 — 开发维护模式）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 保存当前布局 | INSERT / UPDATE | page, element_selector, styles_json, version | 管理员点击保存/自动触发 |
| 导出配置 | SELECT | 全量配置 | 管理员导出JSON |
| 导入配置 | INSERT（批量） | 全部 | 管理员导入JSON |
| 重置为默认 | DELETE | 指定page或全部 | 管理员点击重置 |

### 3.10 AdminSettings（管理员设置）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 验证管理员密码 | SELECT + bcrypt verify | admin_password_hash | 长按激活弹出密码验证 |
| 修改管理员密码 | UPDATE | admin_password_hash | 管理员主动修改 |

## 四、数据库表结构

### 4.1 命名规范

| 规则 | 示例 |
|------|------|
| 表名：小写下划线复数 | `sleep_records`, `daily_scores` |
| 字段名：小写下划线 | `sleep_time`, `overall_score` |
| 主键：`id` UUID类型 | `id UUID PRIMARY KEY DEFAULT gen_random_uuid()` |
| 外键：`{表名}_id` | `user_id UUID REFERENCES users(id)` |
| 时间戳：`created_at` / `updated_at` | 自动填充 |
| 软删除：`deleted_at` | nullable timestamp |
| 索引：`idx_{表}_{字段}` | `idx_sleep_records_user_date` |

### 4.2 所有表总览

| # | 表名 | 说明 | 归属模块 | v1必需 |
|---|------|------|---------|--------|
| 1 | `users` | 用户账户 | 认证 | ✅ |
| 2 | `user_profiles` | 个人资料 | 用户管理 | ✅ |
| 3 | `sleep_records` | 作息记录 | 作息 | ✅ |
| 4 | `diet_records` | 饮食记录 | 饮食 | ✅ |
| 5 | `exercise_records` | 运动记录 | 运动 | ✅ |
| 6 | `stress_records` | 压力记录 | 压力 | ✅ |
| 7 | `risk_records` | 体征记录 | 风险 | ✅ |
| 8 | `daily_scores` | 每日综合评分 | 评分 | ✅ |
| 9 | `layout_configs` | 布局配置 | 开发维护 | ✅ |
| 10 | `admin_settings` | 管理员设置 | 开发维护 | ✅ |
| 11 | `health_tips` | 健康小贴士 | 饮食推荐 | ✅ |

### 4.3 建表语句

```sql
-- ============================================================
-- 1. users 用户账户
-- ============================================================
CREATE TABLE users (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username        VARCHAR(50) NOT NULL UNIQUE,
    password_hash   VARCHAR(255) NOT NULL,
    is_admin        BOOLEAN NOT NULL DEFAULT FALSE,
    created_at      TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    deleted_at      TIMESTAMP WITH TIME ZONE
);
CREATE UNIQUE INDEX idx_users_username ON users(username) WHERE deleted_at IS NULL;

-- ============================================================
-- 2. user_profiles 个人资料
-- ============================================================
CREATE TABLE user_profiles (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID NOT NULL UNIQUE REFERENCES users(id) ON DELETE CASCADE,
    nickname        VARCHAR(50),
    age             SMALLINT CHECK (age >= 0 AND age <= 150),
    gender          VARCHAR(10) CHECK (gender IN ('male', 'female', 'other')),
    height_cm       NUMERIC(5,1) CHECK (height_cm > 0 AND height_cm < 300),
    weight_kg       NUMERIC(5,1) CHECK (weight_kg > 0 AND weight_kg < 500),
    timezone        VARCHAR(50) DEFAULT 'Asia/Shanghai',
    created_at      TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);

-- ============================================================
-- 3. sleep_records 作息记录
-- ============================================================
CREATE TABLE sleep_records (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    record_date         DATE NOT NULL,
    sleep_time          TIME NOT NULL,          -- 入睡时间
    wake_time           TIME NOT NULL,          -- 起床时间
    sleep_quality       SMALLINT CHECK (sleep_quality BETWEEN 1 AND 5),  -- 1-5星
    interruption_count  SMALLINT DEFAULT 0 CHECK (interruption_count >= 0),  -- 中断次数
    score               SMALLINT CHECK (score BETWEEN 0 AND 100),        -- 单次评分
    notes               TEXT,                   -- 备注（预留）
    created_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    -- 同一天只能有一条作息记录
    UNIQUE(user_id, record_date)
);
CREATE INDEX idx_sleep_records_user_date ON sleep_records(user_id, record_date DESC);
CREATE INDEX idx_sleep_records_score ON sleep_records(user_id, score);

-- ============================================================
-- 4. diet_records 饮食记录（多餐次）
-- ============================================================
CREATE TABLE diet_records (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    record_date         DATE NOT NULL,
    meal_type           VARCHAR(10) CHECK (meal_type IN ('breakfast', 'lunch', 'dinner', 'snack')),
    food_description    TEXT NOT NULL,           -- 食物描述
    calories            SMALLINT CHECK (calories >= 0),   -- 估算热量(kcal)
    score               SMALLINT CHECK (score BETWEEN 0 AND 100),  -- 单次评分
    created_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_diet_records_user_date ON diet_records(user_id, record_date DESC);
CREATE INDEX idx_diet_records_meal ON diet_records(user_id, record_date, meal_type);

-- ============================================================
-- 5. exercise_records 运动记录（多次）
-- ============================================================
CREATE TABLE exercise_records (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    record_date         DATE NOT NULL,
    exercise_type       VARCHAR(20) CHECK (exercise_type IN ('running', 'walking', 'cycling', 'fitness', 'ball', 'other')),
    duration_min        SMALLINT NOT NULL CHECK (duration_min > 0 AND duration_min <= 600),
    intensity           VARCHAR(10) CHECK (intensity IN ('low', 'medium', 'high')),
    steps               INTEGER CHECK (steps >= 0),      -- 步数（可选）
    heart_rate          SMALLINT CHECK (heart_rate BETWEEN 30 AND 250),  -- 心率（可选）
    score               SMALLINT CHECK (score BETWEEN 0 AND 100),
    created_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_exercise_records_user_date ON exercise_records(user_id, record_date DESC);

-- ============================================================
-- 6. stress_records 压力记录
-- ============================================================
CREATE TABLE stress_records (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    record_date         DATE NOT NULL,
    stress_level        SMALLINT NOT NULL CHECK (stress_level BETWEEN 1 AND 10),
    anxiety_level       SMALLINT NOT NULL CHECK (anxiety_level BETWEEN 1 AND 10),
    emotion_tag         VARCHAR(20) CHECK (emotion_tag IN (
        'happy', 'calm', 'anxious', 'tired', 'irritable', 'down', 'nervous', 'relaxed'
    )),
    score               SMALLINT CHECK (score BETWEEN 0 AND 100),
    created_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    UNIQUE(user_id, record_date)
);
CREATE INDEX idx_stress_records_user_date ON stress_records(user_id, record_date DESC);

-- ============================================================
-- 7. risk_records 体征记录
-- ============================================================
CREATE TABLE risk_records (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    record_date         DATE NOT NULL,
    systolic_bp         SMALLINT CHECK (systolic_bp BETWEEN 60 AND 250),          -- 收缩压
    diastolic_bp        SMALLINT CHECK (diastolic_bp BETWEEN 30 AND 150),         -- 舒张压
    heart_rate          SMALLINT CHECK (heart_rate BETWEEN 30 AND 250),           -- 静息心率
    blood_glucose       NUMERIC(4,1) CHECK (blood_glucose >= 1.0 AND blood_glucose <= 30.0),  -- 血糖(mmol/L)
    waist_cm            NUMERIC(5,1) CHECK (waist_cm > 0 AND waist_cm < 200),     -- 腰围
    cholesterol         NUMERIC(4,1) CHECK (cholesterol >= 0 AND cholesterol <= 20.0),  -- 胆固醇(mmol/L)
    score               SMALLINT CHECK (score BETWEEN 0 AND 100),
    created_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    UNIQUE(user_id, record_date)
);
CREATE INDEX idx_risk_records_user_date ON risk_records(user_id, record_date DESC);

-- ============================================================
-- 8. daily_scores 每日综合评分
-- ============================================================
CREATE TABLE daily_scores (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    score_date          DATE NOT NULL,
    sleep_score         SMALLINT CHECK (sleep_score IS NULL OR (sleep_score BETWEEN 0 AND 100)),
    diet_score          SMALLINT CHECK (diet_score IS NULL OR (diet_score BETWEEN 0 AND 100)),
    exercise_score      SMALLINT CHECK (exercise_score IS NULL OR (exercise_score BETWEEN 0 AND 100)),
    stress_score        SMALLINT CHECK (stress_score IS NULL OR (stress_score BETWEEN 0 AND 100)),
    risk_score          SMALLINT CHECK (risk_score IS NULL OR (risk_score BETWEEN 0 AND 100)),
    overall_score       SMALLINT CHECK (overall_score IS NULL OR (overall_score BETWEEN 0 AND 100)),
    score_level         VARCHAR(10) GENERATED ALWAYS AS (
        CASE
            WHEN overall_score >= 85 THEN 'excellent'
            WHEN overall_score >= 70 THEN 'good'
            WHEN overall_score >= 60 THEN 'average'
            WHEN overall_score IS NOT NULL THEN 'poor'
            ELSE NULL
        END
    ) STORED,
    created_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    UNIQUE(user_id, score_date)
);
CREATE INDEX idx_daily_scores_user_date ON daily_scores(user_id, score_date DESC);
CREATE INDEX idx_daily_scores_overall ON daily_scores(user_id, overall_score);

-- ============================================================
-- 9. layout_configs 布局配置（开发维护模式）
-- ============================================================
CREATE TABLE layout_configs (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    page                VARCHAR(50) NOT NULL,           -- 页面路由
    element_selector    VARCHAR(255) NOT NULL,          -- CSS选择器
    styles_json         JSONB NOT NULL DEFAULT '{}',    -- 样式参数
    version             INTEGER NOT NULL DEFAULT 1,     -- 版本号
    is_active           BOOLEAN NOT NULL DEFAULT TRUE,  -- 是否为当前生效配置
    created_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    -- 同页面同元素只能有一个版本生效
    UNIQUE(user_id, page, element_selector)
);
CREATE INDEX idx_layout_configs_user_page ON layout_configs(user_id, page);

-- ============================================================
-- 10. admin_settings 管理员设置
-- ============================================================
CREATE TABLE admin_settings (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    admin_password_hash VARCHAR(255) NOT NULL,       -- 管理员密码hash
    is_enabled          BOOLEAN NOT NULL DEFAULT TRUE,
    created_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);
-- 限制只有一条记录
CREATE UNIQUE INDEX idx_admin_settings_singleton ON admin_settings((TRUE));

-- ============================================================
-- 11. health_tips 健康小贴士（饮食页推荐文案）
-- ============================================================
CREATE TABLE health_tips (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    category            VARCHAR(20) NOT NULL CHECK (category IN ('diet', 'sleep', 'exercise', 'stress', 'risk', 'general')),
    title               VARCHAR(100) NOT NULL,
    content             TEXT NOT NULL,
    is_active           BOOLEAN NOT NULL DEFAULT TRUE,
    display_order       SMALLINT NOT NULL DEFAULT 0,
    created_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_health_tips_category ON health_tips(category, display_order);
```

## 五、字段类型规范

| 后端类型 | PostgreSQL | SQLite | 说明 |
|---------|-----------|--------|------|
| 主键 | UUID | TEXT | 使用 `gen_random_uuid()` |

# backend/scores/calculator.py  后续实现
class ScoreCalculator:
    """各维度评分计算器"""

    @staticmethod
    def calc_sleep(record: dict) -> int:
        score = 0
        # 入睡时间
        if record['sleep_time'] <= '23:00': score += 10
        elif record['sleep_time'] <= '24:00': score += 5
        else: score -= 5
        # 睡眠时长（从入睡到起床）
        duration = calc_duration(record['sleep_time'], record['wake_time'])
        if 7 <= duration <= 9: score += 40
        elif 6 <= duration < 7: score += 20
        else: score -= 10
        # 睡眠质量
        score += record['sleep_quality'] * 8  # 1-5星映射到0-40分
        # 中断次数
        if record['interruption_count'] == 0: score += 20
        elif record['interruption_count'] == 1: score += 15
        elif record['interruption_count'] == 2: score += 5
        return max(0, min(100, score))

    # ... 其他维度同理
