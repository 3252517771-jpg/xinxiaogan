## 三、模块边界与职责划分

### 3.1 架构分层

```
┌─────────────────────────────────────────────┐
│  Pages（页面层）                              │
│  职责：路由节点，组合features组装完整页面       │
│  依赖：features/*, components/layout/*       │
│  注意：页面不写业务逻辑，只做组合编排           │
├─────────────────────────────────────────────┤
│  Features（业务层）                            │
│  职责：页面内可复用的业务模块                  │
│  依赖：components/*, hooks/*, services/*     │
│  注意：一个feature只做一件事，允许跨页面复用     │
├─────────────────────────────────────────────┤
│  Components（UI层）                           │
│  职责：通用的UI组件 + 背景层 + IP + 图表       │
│  依赖：types/*, config/*, styles/*           │
│  注意：纯展示，0业务逻辑，通过props驱动         │
├─────────────────────────────────────────────┤
│  Services / Store / Hooks（数据层）            │
│  职责：API请求、状态管理、自定义Hook            │
│  依赖：types/*, config/*                      │
│  注意：不与UI直接耦合，只暴露数据和操作函数      │
├─────────────────────────────────────────────┤
│  Utils / Config / Types（基础层）              │
│  职责：工具函数、配置、类型定义                │
│  依赖：无（纯函数/纯类型）                     │
│  注意：不允许依赖上层模块                       │
└─────────────────────────────────────────────┘
```

### 3.2 依赖规则（反向禁止）

```
Pages → Features → Components → 基础层
Pages → Features → Services/Hooks → 基础层

禁止：Components 依赖 Features
禁止：Services 依赖 Components
禁止：基础层 依赖任何上层
```

### 4.2 路由守卫

```typescript
// src/App.tsx 中
<AuthGuard>
  <AnimatePresence mode="wait">
    <Routes location={location}>
      <Route path="/" element={<HomePage />} />
      <Route path="/sleep" element={<SleepPage />} />
      {/* ...其它页面 */}
      <Route path="/profile" element={<ProfilePage />} />
    </Routes>
  </AnimatePresence>
</AuthGuard>
```

- **AuthGuard**：检查 JWT token 是否存在
  - 无 token → 显示首页但**锁定滚动**，仅展示标语+导航栏，弹出登录模态
  - 有 token → 正常渲染首页全部内容
- **登录模态**：全局 Context 控制，任何地方可触发 `openLoginModal()`

## 六、核心接口协议

### 6.1 API请求封装

```typescript
// src/services/api.ts
const BASE_URL = import.meta.env.DEV ? '/api' : '/api';

export async function request<T>(endpoint: string, options?: RequestInit): Promise<T> {
  const token = localStorage.getItem('token');
  const res = await fetch(`${BASE_URL}${endpoint}`, {
    ...options,
    headers: {
      'Content-Type': 'application/json',
      ...(token ? { Authorization: `Bearer ${token}` } : {}),
      ...options?.headers,
    },
  });
  if (!res.ok) throw new Error(await res.text());
  return res.json();
}
```

## 八、背景层与IP视频状态机实现方案

### 8.1 BackgroundLayer 层级

```
z-0: <video> IP场景视频（全屏固定定位，object-fit:cover）
     兜底：<img> 静态场景图
z-1: <div> 暗角遮罩 (radial-gradient)
z-2: <video> 过渡视频（页面切换时临时显示，播完销毁）
z-3: <div> IP热点区域（5个/28，仅首页激活）
z-10: 前景UI（页面内容）
```

### 8.2 IPVideoPlayer 核心逻辑

```typescript
// src/hooks/useVideoState.ts
function useVideoState(ipId: string, score: number) {
  const ip = IP_CHARACTERS[ipId];
  const [state, setState] = useState<VideoState>('loading');
  const [currentSrc, setCurrentSrc] = useState('');
  
  useEffect(() => {
    // 根据score确定状态
    if (score >= 80) {
      setState('good');
      setCurrentSrc(assetUrl(ip.goodVideo));
    } else if (score >= 60) {
      setState('calm');
      setCurrentSrc(assetUrl(ip.goodVideo)); // 同一视频+CSS滤镜
    } else {
      setState('bad');
      setCurrentSrc(assetUrl(ip.badVideo));
    }
  }, [score, ipId]);

  const triggerFeedback = () => {
    setState('feedback');
    setCurrentSrc(assetUrl(ip.feedbackVideo));
    // 3s后恢复评分态
    setTimeout(() => {
      setCurrentSrc(score >= 80 ? assetUrl(ip.goodVideo) : assetUrl(ip.badVideo));
    }, 3000);
  };

  return { state, currentSrc, triggerFeedback };
}
```
