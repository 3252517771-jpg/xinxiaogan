## 一、项目立项说明

### 1.1 项目背景

中国一二线城市上班族和学生党普遍面临以下健康痛点：

| 痛点 | 表现 |
|------|------|
| 作息紊乱 | 熬夜、睡眠不足、生物钟混乱 |
| 饮食紊乱 | 外卖为主、三餐不定时、营养不均衡 |
| 缺乏运动 | 久坐、缺乏规律运动习惯 |
| 心理压力 | 职场/学业焦虑、情绪管理困难 |
| 慢性病风险 | 高血压、颈椎腰椎、BMI异常等隐患 |

现有健康类产品多偏向冷冰冰的数据工具型，缺乏情感陪伴和持续监督能力，用户很难坚持使用。

### 1.2 项目定位

**"小心肝"** 是一个智能健康陪伴式网页系统，以森林世界观 + 5个IP角色的形式，通过全屏沉浸式视频背景、互动式健康记录、IP角色状态反馈，为一二线城市年轻用户提供**有温度的、陪伴式的健康监督体验**。

核心理念：**Your health, the forest knows.**

### 1.3 目标用户

- **第一期**：一二线城市上班族 + 学生党（通用方案起步）
- **后期分化**：根据使用数据分化上班族/学生党的个性化方案

### 1.4 项目愿景

短期：成为一个让人愿意每天打开的"健康小管家"
长期：通过机器学习模型实现个性化健康风险预测与干预建议

## 二、第一版要做什么（MVP v1）

### 2.1 核心原则

- **前端优先**：先搭全局框架，Mock数据层，逐页开发UI/UX
- **功能淡化**：7个页面全做，但每个页面核心功能轻度
- **交互完整**：保持完整的交互逻辑和数据流转路径
- **IP视频状态机一期落地**：视频过渡、好状态/差状态循环、反馈触发

### 2.2 8个页面/模态范围

| 页面 | 路由 | 核心内容 |
|------|------|---------|
| 登录/注册模态 | — | 用户名+密码登录/注册，JWT认证，未登录锁定首页 |
| 首页 Dashboard | `/` | 综合评分、标语、5维度卡片(ScrollStack)、趋势图、底部个人入口 |
| 作息分析 | `/sleep` | 作息录入、分析摘要、睡眠趋势、IP小恐龙 |
| 饮食分析 | `/diet` | 每日推荐、饮食录入、营养分析摘要、IP龙猫 |
| 运动分析 | `/exercise` | 运动录入、分析摘要、运动趋势、IP小怪兽 |
| 压力/心理 | `/stress` | 压力自评、分析摘要、个性化建议、压力趋势、IP无翼鸟 |
| 健康风险 | `/health-risk` | 体征录入、风险雷达图、风险趋势、分析摘要、IP两脚兽 |
| 个人中心 | `/profile` | 个人资料编辑、历史记录、数据导出、API预留 |

### 2.3 第一期不做的事

- ❌ 智能设备对接
- ❌ 多用户并发/高可用架构
- ❌ 移动端适配

> 🆕 v2.0 更新：推送通知、ML 风险预测、LLM 个性化建议已在 v2.0「AI 陪伴版」中实施，不再属于第一期不做清单。

## 四、产品流程和用户路径

### 4.1 用户首次访问路径

```
打开网页
  → 全屏森林视频背景 + PillNav导航栏（左：小心肝/中：欢迎提示/右：设置）
  → 页面锁定，无法滚动
  → 点击登录按钮 → 弹出登录模态（glass-strong居中浮层）
    ├─ 已有账号 → 输入用户名+密码 → JWT → 登录成功
    └─ 没有账号 → 点击"注册" → 用户名+密码+确认密码 → 注册完成 → 自动登录
  → 页面解锁 → 首页完整内容出现
  → 导航栏评分数字出现（GSAP CountUp 0→82）
```

### 4.2 首页用户路径

```
登录后：PillNav显示 "小心肝 / 欢迎提示 / 评分82 / ⚙️"
   ↓
左半屏："The forest knows your health."（SplitText 入场）
  ↓ （FadeContent 淡出，随着滚动）
中间留白（大片森林视频背景）
  ↓ 往下滚动
ScrollStack 逐张展示5张 MergedShape 卡片（281×177，维度名+状态色点标签）
  ↓ 点击卡片 → GSAP Timeline + 过渡视频(5s) → 进入对应详情页
  ↓ 继续滚动
BorderGlow 趋势图卡片（Recharts 面积图，近7天综合评分）
  ↓ 点击维度标签可切换（综合/作息/饮食/运动/压力/风险）
  ↓
底部 👤 个人中心（玻璃pill，点击跳转/profile）
```

### 4.3 详情页用户路径（以作息页为例）

```
首页点击"作息"卡片
  → GSAP卡片按下动画(0.1s) + 前景UI淡出(0.3s)
  → 过渡遮罩显示 → 播放 T-01 过渡视频(5s)
  → 背景切换至小恐龙场景图
  → 前景UI淡入 → 作息页内容呈现
  → IP视频根据当前评分自动播放对应状态

用户录入睡眠数据:
  → 填写4个字段（入睡时间/起床时间/睡眠质量⭐/中断次数）
  → 点击 [记录今日]
  → 按钮loading → Mock保存 → Toast "✅ 已记录"
  → 背景层小恐龙播放 S-04 反馈视频(3s)
  → 评分重新计算 → IP切换到对应评分状态

页面底部 GooeyNav:
  💤 🍎 💪 💭 ❤️  ← 点击切换详情页（触发过渡视频）
```

### 4.4 IP视频状态机流程

```
各个详情页统一逻辑:

页面加载 → 评分评估 (score)
  ├─ ≥ 80 → 播放好状态循环视频 (5s loop)
  ├─ 60-79 → 播放好状态视频 + CSS滤镜 (brightness 0.85 saturate 0.7)
  └─ < 60 → 播放差状态视频 (3s 单次，定格最后一帧)
     ↓ 用户提交数据
  ✨ 反馈视频 (3s) → 播放完 → 自动恢复评分对应状态

视频切换动画: opacity cross-fade 0.3s
```

### 4.5 页面间导航

| 动作 | 效果 |
|------|------|
| 首页卡片点击 | GSAP Timeline + 过渡视频 (5s) |
| GooeyNav 底部导航 | 点击切换详情页，过渡视频+场景切换 |
| 详情页 `← 返回首页` | 直接切场景图，不播放过渡视频 |
| 导航栏 `⚙️` | 跳转个人主页 `/profile` |
| 底部 `👤 个人中心` | 跳转 `/profile` |

## 六、核心业务对象与数据库设计

> 完整的数据库设计（含11张表完整建表SQL、字段类型规范、评分计算公式）详见 `docs/数据库设计.md`，此处为摘要。

### 6.1 数据库选型

| 环境 | 数据库 |
|------|--------|
| 开发 | SQLite |
| 生产 | **PostgreSQL 15+** |

通过 SQLAlchemy ORM 实现开发/生产无缝切换。

### 6.2 业务对象全景

```
┌──────────┐    1      1..*  ┌──────────────┐
│   User   │────────────────▶│  UserProfile  │
│  (账户)   │                │  (个人资料)    │
└────┬─────┘                 └──────────────┘
     │
     ├──1→*── SleepRecord     → 作息
     ├──1→*── DietRecord      → 饮食
     ├──1→*── ExerciseRecord  → 运动
     ├──1→*── StressRecord    → 压力
     ├──1→*── RiskRecord      → 体征
     ├──1→*── DailyScore      → 综合评分
     ├──1→*── LayoutConfig    → 布局配置
     └──1→1── AdminSettings   → 管理员设置
```

### 6.3 表总览（11张）

| # | 表名 | 说明 | 归属模块 |
|---|------|------|---------|
| 1 | `users` | 用户账户 | 认证 |
| 2 | `user_profiles` | 个人资料（昵称/年龄/身高/体重等） | 用户管理 |
| 3 | `sleep_records` | 作息记录（入睡/起床/质量/中断） | 作息 |
| 4 | `diet_records` | 饮食记录（餐次/食物描述/热量） | 饮食 |
| 5 | `exercise_records` | 运动记录（类型/时长/强度/步数/心率） | 运动 |
| 6 | `stress_records` | 压力记录（压力/焦虑/情绪标签） | 压力 |
| 7 | `risk_records` | 体征记录（血压/心率/血糖/腰围/胆固醇） | 风险 |
| 8 | `daily_scores` | 每日综合评分（含generated score_level） | 评分 |
| 9 | `layout_configs` | 布局配置（JSONB存储样式参数） | 开发维护 |
| 10 | `admin_settings` | 管理员密码设置 | 开发维护 |
| 11 | `health_tips` | 健康小贴士文案（饮食推荐轮换） | 饮食 |

### 6.4 各维度评分规则摘要

> 完整评分计算代码见 `docs/数据库设计.md` 第九章。

| 维度 | 评分维度 | 总分 |
|------|---------|------|
| 作息 | 入睡时间(10)+睡眠时长(40)+睡眠质量(40)+中断次数(20) | 110→100封顶 |
| 饮食 | 三餐完整度(40)+热量合理1600-2200kcal(40)+食物多样性(20) | 100 |
| 运动 | WHO日达标30min(40)+强度权重+步数>8000(30)+心率正常(30) | 100 |
| 压力 | 压力1-3级(40)+焦虑1-3级(30)+正向情绪(30) | 100（反向） |
| 风险 | 6项体征各15-20分，按医学参考范围判定 | 100 |

综合分 `overall_score = AVG(各维度score)`，仅计算有数据的维度。

### 6.5 IP视频状态

| 状态 | 触发评分 | 视频源 | 时长 | 播放方式 |
|------|---------|--------|------|---------|
| 开心 🟢 | score ≥ 80 | X-01.mp4 | 5s | loop循环 |
| 平静 🟡 | 60 ≤ score < 80 | X-01.mp4 + CSS滤镜 | 5s | loop循环 |
| 担心 🔴 | score < 60 | X-03.mp4 | 3s | 单次→定格 |
| 反馈 ✨ | 提交数据后 | X-04.mp4 | 3s | 单次→恢复评分态 |

> X = S(作息) D(饮食) E(运动) P(压力) R(风险)

| 小数 | NUMERIC(M,N) | REAL |
| 日期 | DATE | TEXT (YYYY-MM-DD) |
| 时间 | TIME | TEXT (HH:MM) |
| JSON | JSONB | TEXT |

## 七、开发维护模式（布局编辑器）

### 7.1 激活方式

个人主页 → 长按"小心肝"品牌文字3秒 → 弹出管理员密码验证 → 验证通过进入编辑模式 → 顶部出现 `[🛠 编辑模式已激活]` 指示条 + 右侧出现控制面板

### 7.2 编辑范围

- **页面**：所有页面均可编辑（首页+5详情页+个人主页）
- **粒度**：元素级（位置/宽高/字体大小/字重/颜色/间距/边距/圆角等所有CSS属性）
- **操作方式**：
  - 鼠标点击选中目标组件 → 组件高亮边框
  - 鼠标拖拽移动位置
  - 右侧控制面板显示所有可调CSS参数，用滑块/输入框/颜色选择器调节
  - 框选四角拖拽改变大小

### 7.3 参数保存

- 调节完毕自动保存至 **JSON配置文件**
- 可手动 **导出为JSON** 分享给团队
- 可 **加载JSON** 恢复配置
- 编辑模式下可一键 **重置为默认布局**

### 7.4 退出方式

- 再次点击 `[🛠 退出编辑]` 或关闭控制面板
- 所有调节参数不丢失，持久化至导出JSON
